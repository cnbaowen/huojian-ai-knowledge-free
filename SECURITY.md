# 安全政策

## 支持的版本

安全更新优先提供给最新发布版本。尚未发布正式版本时，以 `main` 分支的最新代码为准。建议及时升级，并在升级前做好数据备份。

## 私密报告漏洞

请进入仓库的 **Security → Advisories** 页面，点击 **Report a vulnerability**，通过 GitHub Private vulnerability reporting 私密提交。不要使用公开 Issue 报告尚未修复的漏洞。

报告中请尽量提供受影响版本、复现步骤、可能影响以及必要的日志或最小样例。请勿附带真实业务数据、访问令牌或模型密钥，也不要在问题修复前公开可利用细节。

## 部署安全建议

- 首次部署时，请更换 `.env.example` 中的全部占位密码和访问令牌。
- 请勿将 `.env`、模型密钥、数据库、上传文件或日志提交到公共代码仓库。
- 建议将系统部署在企业内网、VPN 或受控的反向代理之后。
- 不要将后端服务端口直接暴露到公网，并根据实际环境配置防火墙和访问控制。
