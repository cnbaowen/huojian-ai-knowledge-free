# 安全政策

## 支持范围

安全修复只覆盖最新发布版本。生产部署应使用最新版本，并及时安装安全更新。

## 私密报告漏洞

请优先使用 GitHub 仓库的 Private vulnerability reporting。若仓库尚未开启该功能，请通过项目所有者在仓库主页公布的安全邮箱私下报告。

报告至少应包含：受影响版本、复现步骤、影响范围、日志或最小样例。请勿提交真实客户数据、访问令牌或模型密钥，也不要先公开可利用细节。

## 密钥与数据

- `.env`、模型密钥、数据库、上传文件和日志不得提交到 Git。
- API 只返回“是否配置密钥”，不返回密钥原文。
- 演示数据必须是虚构内容。
- 公开部署前必须更换 `.env.example` 中全部占位值，并限制 18000 后端端口的公网访问。

## 安全边界

本免费版不包含商业版客户服务、CRM、内容发布、情报、智能体、许可证或系统升级模块。发现相关模块、路由或商业数据残留时，应按安全问题处理。
