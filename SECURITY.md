# 安全政策

## 支持范围

安全修复优先覆盖 `main` 分支当前版本和最新 GitHub Release（如有）。生产部署应及时更新，并在升级前做好数据备份。

## 私密报告漏洞

请进入仓库的 **Security → Advisories** 页面，点击 **Report a vulnerability**，通过 GitHub Private vulnerability reporting 私密提交。不要使用公开 Issue 报告尚未修复的漏洞。

报告至少应包含：受影响版本、复现步骤、影响范围、日志或最小样例。请勿提交真实客户数据、访问令牌或模型密钥，也不要先公开可利用细节。

## 密钥与数据

- `.env`、模型密钥、数据库、上传文件和日志不得提交到 Git。
- API 只返回“是否配置密钥”，不返回密钥原文。
- 演示数据必须是虚构内容。
- 公开部署前必须更换 `.env.example` 中全部占位值，并限制 18000 后端端口的公网访问。

## 安全边界

本项目聚焦知识管理和内部知识问答，不包含客户服务、CRM、内容发布、情报、智能体、许可证或系统升级模块。发现项目范围外模块、路由、客户数据或敏感配置残留时，应按安全问题处理。
