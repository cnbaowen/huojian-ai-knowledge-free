# 火建AI知识管理免费版 v0.3.0-rc.3

这是首个提供完整下载页面的公开预发布版，适合希望在自己的服务器或企业内网中快速搭建知识库的团队试用和验证。

系统把文档上传、内容解析、知识切片、检索问答、引用溯源和质量评测串成一个完整流程。资料、索引和配置由部署者自己管理；默认本地抽取模式无需接入外部大模型，也可以体验完整的知识管理和检索流程。

> 当前版本为预发布版，建议先用于体验、测试和方案验证。用于正式业务前，请完成安全配置、数据备份和实际环境验收。

## 主要功能

- **知识文档管理**：上传并解析 TXT、Markdown、DOCX、XLSX 和可复制文字的 PDF。
- **知识切片与检索**：自动切片、建立本地索引，并支持关键词与向量混合检索。
- **内部知识问答**：回答展示引用来源，可返回原文核对，减少无依据回答。
- **知识质量检查**：检查文档解析和切片状态，快速发现无法检索或内容异常的资料。
- **RAG 评测**：通过测试问题观察召回结果和引用情况，帮助持续优化知识库。
- **企业微信群机器人**：配置 Webhook 后，可将知识问答能力接入企业微信群。
- **独立部署**：Docker Compose 启动前端、后端、MySQL 和 Qdrant，默认端口为 `18080` 和 `18000`。

## Docker 部署

要求：Docker Engine 24+、Docker Compose v2+，建议至少 2 核 CPU、4 GB 内存和 5 GB 可用磁盘。

```bash
git clone https://github.com/cnbaowen/huojian-ai-knowledge-free.git
cd huojian-ai-knowledge-free
git checkout v0.3.0-rc.3
cp .env.example .env
```

编辑 `.env`，至少设置 `APP_KEY`、`DB_PASSWORD`、`DB_ROOT_PASSWORD` 和 `FREE_API_TOKEN`，然后启动：

```bash
docker compose up -d --build
docker compose exec backend php artisan migrate --force
```

浏览器打开 `http://localhost:18080`，输入 `.env` 中的 `FREE_API_TOKEN` 登录。

国内网络也可以从 Gitee 克隆：

```bash
git clone https://gitee.com/xiaoguozhou/huojian-ai-knowledge-free.git
```

更完整的环境要求、密钥生成和停止服务方法，请查看[安装说明](https://github.com/cnbaowen/huojian-ai-knowledge-free/blob/v0.3.0-rc.3/docs/INSTALL.md)。

## 已知限制

- 扫描版或纯图片 PDF 暂不包含 OCR，需要先转换为可复制文字的 PDF。
- 默认本地抽取模式侧重流程体验和可控检索，不具备外部大模型的语言生成能力。
- 接入 OpenAI 兼容模型后，问题和召回的知识片段会发送给所配置的模型服务，请根据资料敏感程度选择服务商和部署方式。
- 当前采用部署级访问令牌，适合单个团队在内网或受控网络中使用；暂不提供组织架构和细粒度角色权限。
- 企业微信群机器人依赖可访问的 Webhook；请勿把 Token、模型密钥、Webhook 或后端 `18000` 端口直接暴露到公网。
